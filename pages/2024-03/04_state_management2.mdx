# React State Management

## 状態管理のイマ

### そもそもReactはなぜ状態管理をグローバルに状態管理をしたいのか？
- ReactがUIライブラリであり、クライント上ですべて実行されるため
- ユーザーインタラクション、CRUD処理など入ってくる情報を1箇所で調整する他なかった

### メタフレームワーク(NextJS等)の登場
- NextJSをはじめとするReactメタフレームワークのバックエンドとフロントエンドの両方を持つ(フルスタックWebフレームワーク)
- つまり、状態管理する場所が2箇所ある
- 例えば、サーバー上でDBアクセスをして、状態管理することなく直接レンダリングすることが可能
- Reactは本来の役割であるクライントでのインタラクションに専念できる
- ページごとに独立したシンプルな状態管理(カウンター、トグル、ドラッグアンドドロップ...)をするだけでよい
- 複雑な状態管理は不要で`useState`, `useContext`, `useReducer`くらいで事足りる

### URLを利用して状態管理
- サーバー側ではなくURLにも状態管理を委託する
1. ローカルメモリのプレッシャーを減らす
2. URLをコピペすれば、状態の保存ができる
