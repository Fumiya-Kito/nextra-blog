### 基本的なアルゴリズム一覧（重要度・頻出度付き）

| カテゴリ         | アルゴリズム名                 | 説明                                                           | 重要度・頻出度 |
|------------------|------------------------------|--------------------------------------------------------------|--------------|
| **ソート**       | バブルソート                   | 各要素を隣接要素と比較し、順序が正しくなるように交換する基本的なソート方法。 | 低          |
|                  | クイックソート                 | 分割統治法を用いた高速なソートアルゴリズム。                        | 高          |
|                  | マージソート                   | 分割統治法に基づく安定したソートアルゴリズム。                        | 中          |
|                  | ヒープソート                   | ヒープデータ構造を利用したソートアルゴリズム。                        | 中          |
| **探索**         | 二分探索                       | ソートされた配列に対して効率的に探索を行うアルゴリズム。              | 高          |
|                  | 深さ優先探索 (DFS)             | グラフや木を深く探索するアルゴリズム。                            | 高          |
|                  | 幅優先探索 (BFS)               | グラフや木を広く探索するアルゴリズム。                            | 高          |

|                  | ベルマンフォード法             | 負の重みを持つグラフの単一始点最短経路を求めるアルゴリズム。         | 中          |

|                  | クラスカル法                   | 最小全域木を求めるアルゴリズム。                                  | 中          |

|                  | 繰り返し二乗法                 | 高速にべき乗を計算するアルゴリズム。                              | 中          |

## 解説
### ソートアルゴリズム：
クイックソートは非常に高速であり、実用的な問題で頻繁に使われます。
バブルソートは教育的な目的で紹介されることが多く、実際の競技プログラミングではあまり使用されません。
### 探索アルゴリズム：
二分探索、DFS、BFSは非常に基本的であり、多くの問題で使用される頻度が高いです。
### グラフ理論：
ダイクストラ法は最短経路問題で非常に重要です。フロイドワーシャル法やベルマンフォード法も頻出ですが、やや複雑です。
### 動的計画法：
ナップサック問題や最長増加部分列（LIS）は、動的計画法の典型的な例として多くの問題で登場します。
### 数学的問題：
エラトステネスの篩やユークリッドの互除法は、素数問題や最大公約数問題でよく使用されます。



## 問題文から適切なアルゴリズムを判断するコツ
### 問題文をしっかり読む：

問題の全体像を把握し、何を求められているかを理解する。
入力の制約を確認し、大規模データや特定の条件があるかを確認する。
### キーワードを見つける：
特定のキーワード（例：最短経路、全探索、ソート、部分和など）がある場合、そのキーワードに関連するアルゴリズムを考える。

### 問題のタイプを特定する：
問題がグラフ理論、動的計画法、探索、ソート、数学的問題などのどのカテゴリに属するかを考える。


### 例やヒントを参考にする：
提供されている例やヒントから、どのようなアルゴリズムが適用できるかを推測する。

### 類似問題の経験：
以前に解いたことのある類似問題を思い出し、そのとき使ったアルゴリズムを参考にする。
