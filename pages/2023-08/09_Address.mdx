## アドレス
昨日、Goのポインタを勉強したので、前提知識の復習

### 高水準言語（高級言語）
- 低水準言語(機械語orアセンブリ言語)＝CPUが直接解釈・実行できる。
    - 低い＝ハードウェアに近い
- 高水準言語＝抽象的な語彙・記法・構文を用いて処理を記述することができる。
    - 高い＝自然言語に近い
    - ソフトウェア開発に向く
    - **実行するには機械語へ翻訳が必要**

### コンパイラとインタープリタ
- コンパイラ: 開発時にまとめて変換(**コンパイル**)する＜翻訳者の意＞
- インタープリタ: 実行時に逐次変換しながら並行して実行する＜通訳者の意＞

| |メリット|デメリット|言語の例|
| :--- | :--- | :--- | :--- |
| コンパイラ型 | プログラムの実行速度が速い | 実行に手間がかかる <br/> エラーが見つけにくい | **Go** <br/> Java <br/> C |
| インタープリタ型 | プログラムをすぐ実行できる <br/> エラーを見つけやすい | プログラムの実行速度が遅い |Ruby <br/> JS <br/> Python |

※変換されて「実行できるような形」とはバイナリーコードの実行ファイルを指す


### 変数とメモリとアドレス
- メモリは、1バイト毎に番号がつけられ、区別されている
- 変数は実行ファイルになると、番号が割り当てられる
- 変数は、メモリ上の該当の番号の区分に格納され、記憶される
- この変数に付与されるメモリの区分番号をアドレスという

アドレス: `0x1040a0d0` のような16進数で表される
